**Command Center** tracks what you do — habits, exercise, sleep, meals, water, reading, projects, finances, focus sessions. It's your daily operations dashboard.

**The Mind** tracks what you find — articles, code snippets, images, highlights, ideas, screenshots. It's your personal knowledge base with AI-powered search. Like [mymind.com](https://mymind.com), but yours.

**The Extension** lives on every browser tab. Save content to The Mind. Log water, toggle habits, check your capacity — all without leaving the page you're on.

**OpenClaw** is the autonomous agent behind both. It evaluates your standing directives, generates weekly debriefs, surfaces saved knowledge when relevant, and observes patterns across your data.

#### Capacity Meter

A real-time 0-100 score: how much bandwidth do you have right now?

Calculated from signals you already track:

| Signal | Weight | What it measures |
|--------|--------|-----------------|
| **Sleep** | 30% | Last night's hours vs your personal baseline, plus quality |
| **Physical** | 25% | Exercise done today, water intake %, meals tracked |
| **Mental load** | 25% | Pending project tasks, overdue habits, active project count |
| **Momentum** | 20% | 3-day habit completion trend — improving or declining |

Displayed prominently in Today Overview. OpenClaw reads this score and adjusts behavior — more suggestions when you're high capacity, fewer interruptions when you're low.

**Calibration**: First 21 days, you rate your energy 1-5 at end of day. The system adjusts weights to match your subjective experience. After that, it runs on autopilot.

**Database**:
```sql
create table capacity_logs (
  id              uuid primary key default gen_random_uuid(),
  date            date not null unique,
  score           int not null check (score between 0 and 100),
  sleep_score     int,
  physical_score  int,
  load_score      int,
  momentum_score  int,
  self_report     int check (self_report between 1 and 5),
  created_at      timestamptz default now()
);

create table capacity_config (
  id               uuid primary key default gen_random_uuid(),
  sleep_weight     decimal(3,2) default 0.30,
  physical_weight  decimal(3,2) default 0.25,
  load_weight      decimal(3,2) default 0.25,
  momentum_weight  decimal(3,2) default 0.20,
  calibrated       boolean default false,
  calibration_days int default 0,
  updated_at       timestamptz default now()
);
```

**API**: `GET /api/capacity` | `GET /api/capacity/history` | `POST /api/capacity/calibrate` | `GET|PATCH /api/capacity/config`

### Part 2 — The Mind

A separate app. Its own repo, its own URL. Save anything. Find it with natural language. No folders, no tags to manage, no organization burden.

<br />

#### The Product

You come across an article about database sharding. You highlight a paragraph, hit `Cmd+Shift+S`. It's saved. Three weeks later, you type "how to split data across multiple servers" and it comes back — even though you never used those words. That's semantic search.

You right-click an architecture diagram. Saved. You paste a code snippet from Stack Overflow. Saved. You type a random idea at 2am into the command bar. Saved. Everything goes into one flat stream, auto-tagged, auto-summarized, auto-embedded for search.

No folders. No manual tags. No organizational tax.

<br />

#### What You Can Save

| Type | How | What's stored |
|------|-----|---------------|
| **Article** | Extension: save page / paste URL | URL, title, extracted text, auto-summary |
| **Highlight** | Extension: select text, `Cmd+Shift+S` | Selected text, source URL, surrounding context |
| **Image** | Extension: right-click save | Image URL, auto-generated description |
| **Code** | Extension or paste | Code block, detected language, source |
| **Thought** | Command bar or extension | Raw text, timestamp |
| **Screenshot** | Extension: region clip | Image blob, page URL |

<br />

#### The Processing Pipeline

Every save triggers this, automatically:

```
Save item
  │
  ├─→ Extract text (readability algorithm for articles)
  │
  ├─→ Generate summary (2-3 sentences via fast LLM)
  │
  ├─→ Auto-generate tags (from content analysis)
  │
  ├─→ Generate embedding (OpenAI text-embedding-3-small → 1536-dim vector)
  │
  └─→ Store in Supabase + pgvector
```

<br />

#### Search — The RAG Pipeline

You search in natural language. The system doesn't match keywords — it matches meaning.

```
Query: "that article about scaling databases"
  │
  ├─→ Embed the query (same model as storage)
  │
  ├─→ Cosine similarity search (pgvector)
  │
  ├─→ Re-rank results:
  │     60% semantic similarity
  │     25% recency
  │     15% access frequency
  │
  └─→ Return top results with highlighted passages
```

You find things by describing what you remember, not by remembering exact words or which folder you put it in.

<br />

#### The UI

The Mind has its own web app at its own URL. Clean, minimal, fast.

- **Home**: Masonry grid of everything you've saved, newest first
- **Search bar**: Always visible at top. Type naturally. Results update in real-time
- **Filters**: By type (articles, code, images, thoughts), by auto-tag, by date range, by source domain
- **Item detail**: Full content view with related items, metadata, source link
- **Clusters**: Auto-detected topic groups ("You've saved 7 things about Go concurrency")

<br />

#### Auto-Organization

- **Tags**: Generated by LLM at save time. You never touch them.
- **Clusters**: Weekly, the agent groups related items into topic clusters.
- **Temporal links**: Items saved the same week are loosely associated.
- **Source links**: Items from the same domain cluster together.

<br />

#### Agent Integration

OpenClaw connects The Mind to Command Center:

- **Proactive surfacing**: Working on a project? Agent searches The Mind for related saved items and surfaces them.
- **Pattern detection**: "You've saved 5 articles about event sourcing this month. Emerging interest?"
- **Weekly debrief**: "This week you saved 12 items — 7 engineering, 3 health, 2 ideas."
- **Context suggestions**: The browser extension shows a badge when the current page relates to something you've saved before.

#### Tech Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Separate repo or monorepo? | **Monorepo** (same Next.js app, separate route group) | Shared Supabase, shared types, shared auth. Deploy as one app with two entry points, or as separate Vercel projects from the same repo |
| Embedding model | **OpenAI text-embedding-3-small** | Cheap ($0.02/1M tokens), fast, 1536 dimensions, good enough for personal scale |
| Vector store | **pgvector** (Supabase extension) | Already on Supabase. No new infra. IVFFlat index handles thousands of items easily |
| Summarization | **Claude Haiku** or **GPT-4o-mini** | Fast, cheap, good summaries. Called once per save |
| Content extraction | **@mozilla/readability** or **Cheerio** | Extract article text from HTML. Standard approach |

**New dependency**: `pnpm add openai` (for embeddings)

#### Directive System

Standing orders. Persistent instructions the agent continuously evaluates.

Not reminders. Not goals. **Directives** — autonomous rules with trigger conditions, actions, escalation ladders, and success criteria.

```
DIRECTIVE: "If I haven't logged water by 3pm, nudge me"
  Trigger:    State-based — water < 4 AND hour > 15
  Action:     Send notification
  Escalation: After 2 hours, send again with "Your plant is dying"
  Success:    water >= 4
  Status:     ✅ Active — last triggered yesterday, fulfilled
```

```
DIRECTIVE: "Don't let me skip exercise two days in a row"
  Trigger:    Daily at 8pm — check last 2 days of exercise_logs
  Action:     Notification with suggested workout
  Escalation: Next morning, block morning briefing until acknowledged
  Success:    exercise_log exists for today
  Status:     ✅ Active — 89% fulfillment rate (30 days)
```

**Database**:
```sql
create table directives (
  id                uuid primary key default gen_random_uuid(),
  instruction       text not null,
  trigger_type      text not null check (trigger_type in ('time', 'event', 'state')),
  trigger_config    jsonb not null,
  actions           jsonb not null,
  escalation        jsonb,
  success_condition text,
  priority          text default 'normal' check (priority in ('critical', 'normal', 'low')),
  active            boolean default true,
  created_at        timestamptz default now(),
  updated_at        timestamptz default now()
);

create table directive_logs (
  id            uuid primary key default gen_random_uuid(),
  directive_id  uuid not null references directives(id) on delete cascade,
  evaluated_at  timestamptz not null,
  triggered     boolean not null,
  action_taken  text,
  outcome       text,
  fulfilled     boolean,
  created_at    timestamptz default now()
);
```

**API**: `GET|POST|PATCH|DELETE /api/directives` | `POST /api/directives/:id/evaluate` | `GET /api/directives/:id/history` | `GET /api/directives/stats`

**Page**: `/directives` — card per directive, history timeline, trigger builder, performance stats.

#### Agent Console

Everything the agent does, wants to do, or notices — in one feed.

Three streams:

| Stream | What it shows | Your action |
|--------|-------------|-------------|
| **Actions** | Things the agent did autonomously — logged data, generated briefs, evaluated directives | Undo |
| **Approvals** | Things the agent wants to do but needs your OK — add items, adjust schedules, cancel subscriptions | Approve / Reject |
| **Observations** | Patterns the agent noticed — correlations, anomalies, trends | Dismiss / Act on |

**Trust calibration**: The console tracks your approval/rejection rate per action type. Consistently approve article additions? Agent suggests auto-approving them. Consistently reject something? Agent stops asking. Your trust boundaries evolve organically.

**Database**:
```sql
create table console_entries (
  id            uuid primary key default gen_random_uuid(),
  stream        text not null check (stream in ('action', 'approval', 'observation')),
  title         text not null,
  description   text,
  metadata      jsonb,
  status        text default 'active' check (status in ('active','approved','rejected','undone','dismissed')),
  directive_id  uuid references directives(id),
  undo_data     jsonb,
  created_at    timestamptz default now(),
  resolved_at   timestamptz
);
```

**API**: `GET /api/console` | `GET /api/console/pending` | `POST /api/console/entries` | approve/reject/undo/dismiss endpoints

**Page**: `/console` — scrollable feed with stream filter tabs. Mini version in Today view showing pending count + latest entry.

#### Context Layer

Not a page. An intelligence layer woven into existing sections.

OpenClaw cross-references all your data and generates small, dismissible insight cards that appear inline where they're relevant:

- In **habits**: "You complete 91% of habits on days you exercise before noon"
- In **water plant**: "Your intake drops 35% on weekends. It's Saturday."
- In **meals**: "You burned 400 cal this morning. Consider a higher-protein lunch."
- In **projects**: "27 commits, 0 pushes. Might want to push."

**Rules**: Max 3 visible at once. 70% minimum confidence. No repeats of dismissed insights for 7 days. Every insight must be actionable.

**Database**:
```sql
create table context_insights (
  id            uuid primary key default gen_random_uuid(),
  section       text not null,
  content       text not null,
  confidence    decimal(3,2) not null,
  data_sources  text[] not null,
  status        text default 'active' check (status in ('active', 'dismissed', 'expired')),
  dismissed_at  timestamptz,
  expires_at    timestamptz,
  created_at    timestamptz default now()
);
```

**Implementation**: A `<ContextInsight section="habits" />` component dropped into each existing section. Purely additive — zero changes to existing components.

<br />

#### Weekly Debrief

Every Sunday at 6pm, OpenClaw generates an analytical review of your week.

Five sections:

1. **The Numbers** — all metrics vs last week vs 4-week average
2. **Wins** — streaks, milestones, personal records, completions
3. **Patterns** — cross-domain correlations the agent found
4. **Directive Performance** — which standing orders were met, which failed
5. **Next Week** — proposed focus areas and adjustments (you review and confirm)

Uses the existing `briefs` table with type `weekly_debrief`. Generated via API call that compiles data from every table.

**API**: `GET /api/debrief/weekly` | `POST /api/debrief/weekly/generate` | `PATCH /api/debrief/weekly/:id`

## Build Order

<br />

### Phase 1 — Strengthen the Home

Build the features that make Command Center a daily driver worth deploying.

```
1.1  Command Bar          the interaction pattern everything else builds on
1.2  Sleep Tracker         simple CRUD, fills health tracking gap
1.3  Focus Timer           pomodoro + session logging
1.4  Finance Tracker       new page, expenses + budgets + subscriptions
1.5  Capacity Meter        composite score from existing + new data
1.6  Navigation update     add new pages to sidebar, group sections
1.7  Settings page         API key generation for extension + agent
```

**Milestone**: Complete personal OS. Deploy to Vercel.

<br />

### Phase 2 — Build The Mind

The knowledge base. Save anything, find anything.

```
2.1  Database setup        pgvector extension, mind_items table, indexes
2.2  Save API              POST /api/mind/save with full processing pipeline
2.3  Search API            Semantic search with RAG re-ranking
2.4  Web UI                /mind page — grid, search, filters, detail view
2.5  Clusters              Auto-grouping of related items
```

**Milestone**: Working knowledge base at its own URL. Manual save via paste/URL.

<br />

### Phase 3 — Build the Extension

Make both apps accessible from every browser tab.

```
3.1  Extension scaffold    Plasmo setup, manifest, auth flow
3.2  Quick Save            Cmd+Shift+S — save page/selection/image to Mind
3.3  Side Panel            Dashboard glance — capacity, water, habits, agent
3.4  Command Bar overlay   Cmd+Shift+K on any page
3.5  Context badge         Show related saved items for current page
3.6  Auto-capture          Opt-in passive browsing log
```

**Milestone**: Chrome extension sideloaded. Save from anywhere, check dashboard from anywhere.

<br />

### Phase 4 — Agent Intelligence

Transform OpenClaw from a webhook receiver into an autonomous collaborator.

```
4.1  Directive System      standing orders with triggers, actions, escalation
4.2  Agent Console         transparency feed — actions, approvals, observations
4.3  Context Layer         inline insights across existing sections
4.4  Weekly Debrief        Sunday evening analytical review
4.5  Trust calibration     auto-adjust agent autonomy from approval patterns
```

**Milestone**: An AI agent that evaluates your standing orders, surfaces knowledge when relevant, generates weekly analysis, and adapts to your trust boundaries.
